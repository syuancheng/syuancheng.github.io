### 面向对象

#### 1.基本运行原理

- Java源代码（.java)经过java编译器（javac）编译之后，并没有直接转化为机器码，而是转化成一种中间格式，成为字节码(.class)，字节码再经过java虚拟机转化成特定CPU架构的机器码。
- 字节码经过虚拟机的类加载器进行加载，然后解释器逐行对其进行解释执行。
- 引入了JIT编译器：在第一个编译的时候会将字节码和机器码都保存下来，下次就可以直接使用。
- JIT编译器的作用：在程序运行期间，将Java字节码编译成平台相关的二进制代码。正因为此编译行为发生在程序运行期间，所以该编译器被称为Just-In-Time编译器。

#### 2.jre和jdk

- jre是java运行时环境，它包含的是**运行已经编译的java程序所需东西**，包含java虚拟机、java类库、命令。
- jdk是java开发工具包，它包含jre拥有的所有东西，还包含编译器（javac）、和一些工具如javadoc，他可以创建和编译程序。

#### 3.java和c++

- 都是面向对象，支持封装、继承、多态
- java不提供指针
- java是单继承，c++是多继承
- java具有自动内存管理机制

#### 4.面向对象的特性

##### 封装：
- 将对象的属性私有化，并提供可以被外界访问的方法。
- 实现对属性的数据访问限制，同时增加了程序的安全性。通过方法我们可以限制该属性的访问，如果是直接调用属性点话可能会产生非法属性的值。
##### 继承：
在一个类定义的基础上面重新定义另一个类。一般使用extends关键字，这里object是所有类的父类。通过这种方式可以提高代码的复用性。
##### 多态：
> 为什么要有多态？
> 在实际开发中发现类中方法需要进行改进，但是已经提交给了客户使用。在下次创建该类的对象时要对该方法进行改进，但是因为程序的其他部分已经在使用该方法，因此我们需要继承该类并重写父类中需要改进的方法。
- 父类引用指向子类实现。在编译时无法确定引用指向的对象以及可以调用的方法，只有在运行时才能确定。
- 实现必须满足1.继承；2.重写父类方法；3.父类引用指向子类实现。

#### 5.重载和重写：

##### 重载：

发生在一个类中的，方法名相同，参数类型(数量、类型、**顺序**)不同；方法返回值、修饰符也可以不同。（不能仅仅是返回值不同）

##### 重写：

发生在继承了父类的子类当中，方法名、参数列表必须相同，返回值的范围必须小于等于父类。而且如果父类的方法是private，那么子类就不能重写该方法。

#### 6.常用关键字

##### final：

- 变量：如果是基本类型，那么一旦初始化之后就不能再改变，相当于一个常量；如果是引用类型，那么一旦初始化之后就不能指向其他对象，是可以对对象的属性进行修改的。
- 类：修饰一个类，这个类就不可以被继承。
- 方法：修饰一个方法，这个方法就不可以被重写，但是可以被重载。

##### super和this

- super：在构造方法中用来调用父类的构造方法，但是一定要放在首行否则会编译错误。也可以指向父类中的成员变量。
- this：用来修饰变量表示的是当前类中的成员变量；this()表示调用当前类中的其他构造方法

```
private int length;
public Syuan(int id, String name) {
    super(id, name);
}

public Syuan(int id, String name, int length){
    this(id, name);
    this.length=length;
}
```

> **Note:**
>
> 这两个关键词都不可以用在static方法或者代码块中，在静态成员中访问非静态成员之所以不被允许是因为——静态方法在调用前不需要创建任何实例对象，当静态方法调用时，this所指的对象根本没产生。
>
> **ref：**https://blog.csdn.net/wx123456wx/article/details/3118513

##### static：

- 变量：类变量，在类加载的过程中被初始化(类加载的准备阶段会给类变量分配内存并初始化)，存放在方法区中。
- 方法：可以直接通过类名调用
- 初始化块：在初始化快部分介绍
- 内部类：在内部类部分介绍

##### instanceof：

使用： `对象 instanceof 类名`

作用：在进行类型强转时，判定前一个对象是否是后面类的实例，是否可以进行强制转换，从而保证代码更加健壮。

判断对象是不后面的类，或者其子类、实现类的实例对象。如果是true。
- 前面对象的编译时类型要么与后面类相同，要么是后面类的子类，否则会引起编译错误。

#### 7.代码块和构造方法
##### 静态代码块——对类进行初始化
>位置：类成员位置，用static修饰的代码块
>
>作用：对类进行一些初始化，而且只加载一次，当new多个对象时，只有第一次会调用静态代码块，因为，静态代码是属于类的，所有对象共享一份
>
>静态不能访问非静态：因此静态代码块不能初始化实例变量

##### 构造代码块
>位置：类成员的位置，就是类中方法之外的位置
>
>作用：把多个构造方法共同的部分提取出来，共用构造代码块
>
>调用：每次调用构造方法时，都会优先于构造方法执行，也就是每次new一个对象时自动调用，对对象的初始化

##### 构造方法
>作用：对象初始化
>
>构造器可重载
>
>this关键字  优先执行

**执行顺序：静态代码块---构造代码块---构造方法**

#### 8.内部类

应用场景：https://www.cnblogs.com/firefrost/p/5064210.html

##### 为什么要有？

> 更好的封装，内部类隐藏在外部类之中，不允许外部类以外的其他类来访问。适用于那种**只需要一次使用**的类。

##### 成员内部类
1. 外部类可以访问内部类的成员和方法，但是要先创建一个外部类的对象，通过这个对象来使用成员和方法。
2. 内部类也可以访问外部类的成员和方法，但是要注意的是如果外部类和内部类的成员的名字相同的话，会默认使用内部类的成员，这时可以通过`外部类.this.成员变量/方法`的形式来访问。
3. 编译完成之后会生成完全不同的两个类。

##### 静态内部类
1. 内部类当中不能使用外部类的非静态成员或方法。
2. 不需要依赖外部类，她随着外部类的产生而产生。

##### 局部内部类

1. 是定义在方法中的内部类。
2. 对内部类的访问仅仅局限于方法内。

##### 匿名内部类

- 唯一一种没有构造方法的类。

**实现形式**

```
new 实现接口/抽象类(){
	//类体
}
```

#### 9.java类和包
##### .java文件
- 可以有多个类，但只能有一个public类，而且public类的名字要和文件名一致
- 也可以没有public类，这种情况下可以随意命名
- 因为编译器在运行时，针对一个java源代码文件只会接受一个public类，否则会引起错误

```
person p=new person("krystal",20);
```

**这句话发生了什么事情？**

1. 类加载检查
2. 在堆内存中开辟空间，分配内存地址在堆内存中建立对象的特有属性，并进行默认初始化
3. 对属性进行显示初始化
4. 对对象进行**构造代码块**初始化
5. 对对象进行对应的**构造方法**初始化
6. 将内存地址赋给栈内存中的p变量

#### 10.java内存分配机制
##### 成员变量的初始化和内存中的运行机制
系统加载类或者创建该类的实例时，系统会自动为成员变量分配内存空间，并在分配内存空间后为成员变量指定初始值。

**例如**
```
Person p1=new person();
Person p2=new person();
p1.name="krystal";//实例变量
p2.name="amber";
p1.eyeNum=2;//类变量
p2.eyeNum=3;
```

**类的初始化：**
Person p1=new person();是系统第一次使用Person类，那么系统先会加载这个类，然后初始化这个类（***类的初始化就是通过调用静态代码块来初始化类变量***），在**准备**阶段会给类变量eyeNum分配内存并指定默认初始值（0、null）

然后把Person对象赋给p1变量，Person对象包含有name实例变量。实例变量name是在创建实例的时候分配内存并指定初始值。当执行Person p2=new person();语句时就不会再类初始化了。

##### 局部变量
局部变量定义后必须经过显式初始化以后才能被使用，系统不会为局部变量初始化。直到程序为这个变量赋值时才会为其分配内存，并将初始值保存在这块内存中（局部变量不属于任何类，他随着方法的开始而开始结束而结束，因此他总是**保存在所在方法的栈内存**中）。**如果局部变量是基本类型，则直接将值保存在该变量的内存中；如果是引用类型，则存放地址值。**

##### 方法
在调用方法的时候，要给方法中的形参赋值，这个过程叫做值的传递，根据类型不同分为值传递和引用传递。
###### 值传递(基本类型）：
当调用方法时，会在方法中产生所需的局部变量，并且会给他们进行初始化，也即是给他们赋值，这里赋的值是副本，并不是值的本身。

###### 引用传递（类类型）：
当调用方法时，会在方法中产生所需的局部变量，并且会给他们进行初始化，也即是给他们赋值，这里赋的值是对象在堆内存中的地址值（对象的一个引用），因此方法中的操作实际上操作的是所引用的堆中的对象。

#### 11.抽象类和接口
##### 抽象类
1. 抽象类可以有具体的方法（非抽象方法），抽象方法不能有方法体
2. 实例化：当抽象类为父类，他的子类是普通类时，可以将创建的子类实例被父类的引用变量所指向。
3. 不存在类抽象方法——abstract和static不能同时出现，因为抽象方法没有方法体，而被static修饰的静态方法是可以被类名直接调用的。
4. private和abstract不能同时出现，因为抽象方法只有被子类重写才会有方法体，才有意义。而private权限的方法是不能被重写的。

##### 接口
1. 一个接口可以有多个父接口——多继承
2. 接口中没有构造方法和代码块
3. **接口中的所有成员都是public修饰的**
4. **成员变量由public static final修饰**


##### 二者的异同
###### 相同之处
- 都具有抽象方法，而且必须被子类重写
- 必须先接口后父类

###### 不同之处
- 接口中不能包含普通方法；抽象类中可以有
- 接口中只能定义静态常量，不能有普通变量；抽象类中两者都有
- 接口中不含构造器，初始化块；抽象类中有，让子类调用构造器实现抽象类的初始化
- 接口多继承；抽象类单继承
