一般认为new出来的对象都是被分配在堆上，但是这个结论不是那么的绝对，通过对Java对象分配的过程分析，可以知道有两个地方会导致Java中new出来的对象并不一定分别在所认为的堆上。这两个点分别是Java中的**逃逸分析**和**TLAB**（Thread Local Allocation Buffer）。本文首先对这两者进行介绍，而后对Java对象分配过程进行介绍。



逃逸分析是一种算法可以用来计算出new的对象的引用使用的范围从而决定是否要将其分配到堆当中。

可以很好的减少堆内存的分配压力。



```
class Main {  
  public static void main(String[] args) {  
    example();  
  }  
  public static void example() {  
    Foo foo = new Foo(); //alloc  
    Bar bar = new Bar(); //alloc  
    bar.setFoo(foo);  
  }  
}  
 
class Foo {}  
 
class Bar {  
  private Foo foo;  
  public void setFoo(Foo foo) {  
    this.foo = foo;  
  }  
} 
```

可以看到foo和bar对象都没有逃脱example的范围，因此会放在栈当中。



TLAB(每个线程在eden区分配一块内存)，旨在解决内存分配的并发问题。

这里一般会保存那些“用过即丢”的小对象，而且是不需要线程共享的。TLAB是线程私有的，因此不存在锁开销。

这种情况下分配对象的时候不需要锁住整个堆，只需要在自己的缓存区分配就好了。

内核空间和用户空间：

内核空间主要是指用于操作系统运行时所使用（程序调度、虚拟内存、联结硬件资源）

为什么要分成这样？

操作系统运行的稳定性和安全性，运行在操作系统中的用户程序不能使用操作系统的内存空间。

而且如果要访问硬件资源只能由操作系统来发起，程序不可以直接访问硬件资源。对硬件资源的访问成为系统调用，这个要伴随着从内核空间到用户空间的数据复制，很费时间。
